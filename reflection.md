# Reflection

This project fundamentally transformed my understanding of AI systems.

Initially, I assumed that calling an API and providing a simple prompt would suffice to build an effective chatbot. However, the early results proved quite generic and failed to reflect any authentic personality.

The pivotal moment came when I started refining the system prompts. I realized that merely stating "you are this person" is insufficient. The model requires explicit instructions about tone, behavior, and constraints. Adding few-shot examples yielded significant improvements, particularly in ensuring response consistency.

The GIGO principle became abundantly evident during this project. When the prompts were vague, the output remained equally vague. Once I incorporated structure, examples, and explicit instructions, the responses improved markedly.

One compelling challenge involved making the personas feel authentically distinct. Initially, all three sounded remarkably similar. I had to deliberately emphasize differences in tone — making Anshuman more guiding, Abhimanyu more direct, and Kshitij more technical.

Given more time, I would enhance the UI further and possibly implement chat history persistence. I would also experiment with more sophisticated prompt techniques such as dynamic memory or context-aware responses.

Ultimately, this project demonstrated that prompt engineering extends beyond writing instructions — it fundamentally shapes behavior.
